How Job Routing Works

When you submit a job, the ML classifier analyzes the parameters and predicts the job class:

small_quick (low data, low complexity) → Runs locally on this machine
medium_cpu (moderate data/complexity) → Sent to NYU Dataproc YARN cluster (2 executors)
large_intensive (high data/complexity) → Sent to NYU Dataproc YARN cluster (4 executors)

The PySpark job generates synthetic transaction data, runs distributed analytics (category revenue, top customers, time-series, cross-tabs), and saves results.

About YARN

YARN (Yet Another Resource Negotiator) is the resource manager on the NYU Dataproc cluster. When OrcaFlow classifies a job as medium_cpu or large_intensive, it submits the PySpark job to YARN via spark-submit --master yarn on the Dataproc master node. YARN allocates executors across the cluster nodes and manages job lifecycle.

Classification Rules (Fallback)

When no trained model is loaded, rule-based classification is used:
small_quick: dataset < 500MB AND complexity < 4
large_intensive: dataset > 2000MB OR complexity > 7
medium_cpu: everything else
